Features 

  • There is now support multiple protocols in a single node. Protocols may be the same type if they use different TCP port 
  • Support of ports for various protocols. The protocols must implement tid.inet.protocols.Protocol
  • To add a protocol / session bgp different simulation environment must take the following steps: 


    1. class that implements the interface tid.inet.protocols.protocol.java 
    2. add the string identifier in the interface protocol 
    3. in class tid.inet.protocols.Factory.java add the factories for the following classes 


      • GeneralParametersFactory 
      • doConfigFactory 
      • logsFactory 
      • mapStringFactory 


  • Support for multiple interfaces (IP addresses) per node. 
  • Support for multiple nodes per link 
  • The protocol "BGP 4" can now use a port other than the default was. 
  • The protocol "BGP 4" using any of the interfaces of a node (but only one)
  • Events: Only work event "DOWN"
  • The physical topologies now aren't created automatically. They have to be specified in the xml file 
  • Now you can turn on and off the error and debug output from Console to a flag in xml files 
  • XML configuration file. 
  • There are two xml files. one with configuration parameters (ie: test_1_config.xml) and other with simulation (test_1_events.xml) 
  • The configuration file format has changed with new parameters. (see xml example) by adding new features;
